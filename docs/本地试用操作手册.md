# 本地试用操作手册

本文档面向第一次在本地试用墨衡 DocForge v0.1 的用户。

## 一、当前项目是否区分前后端？

当前 v0.1 已经区分前后端和核心业务层：

1. Vue 前端工作台；
2. FastAPI API 层；
3. `docforge_core` 核心业务层；
4. Streamlit 调试入口。

旧版单入口说明已经过期。DocForge v0.1 的正式入口是 Vue3 三栏式 Agent 工作台，路径为 `frontend/docforge-web`。

Streamlit 只作为开发调试入口 / 旧 Demo 入口。

```text
浏览器
  ↓
Vue3 工作台
  ↓ HTTP
FastAPI API
  ↓ Python 调用
docforge_core
  ↓
本地文件 / Qdrant / 外部 LLM API
```

## 二、需要启动哪些服务？

必须启动：

1. FastAPI 服务；
2. Vue 前端服务。

可选启动：

1. Streamlit 调试入口。

不需要启动：

1. 独立 Qdrant Server；
2. Docker；
3. 数据库服务；
4. WebSocket 服务。

Qdrant 使用 Local Persistent Mode，数据目录为 `data/qdrant/`。

## 三、环境准备

需要准备：

1. Python 3.11 或 3.12；
2. Node.js；
3. npm；
4. Corepack；
5. pnpm；
6. 根目录 `.env`；
7. `frontend/docforge-web/.env`。

后端和前端使用不同工具链：

- Python `.venv` 只管理后端依赖；
- Node.js / npm / pnpm 管理 Vue 前端依赖；
- `pnpm` 是 Node.js 包管理器，不属于 Python `.venv`，不要安装到 `.venv`。

检查前端工具链：

```bash
scripts/check_frontend_env.sh
```

也可以手动检查：

```bash
node -v
npm -v
corepack -v
pnpm -v
```

如果 `pnpm` 不存在，先安装系统 Node.js，然后启用 Corepack：

```bash
corepack enable
corepack prepare pnpm@latest --activate
```

macOS 安装 Node.js 的常见方式：

```bash
brew install node
```

不要使用临时 Node 或 `/private/tmp` 下的 Node 启动长期开发环境，避免 Vite / Rollup native 包加载失败。

配置根目录 `.env`：

```bash
cp .env.example .env
```

mock 模式：

```dotenv
DEFAULT_LLM_PROVIDER=mock
DEFAULT_EMBEDDING_PROVIDER=mock
```

真实模式：

```dotenv
DEFAULT_LLM_PROVIDER=qwen
DEFAULT_EMBEDDING_PROVIDER=jina
QWEN_API_KEY=你的_Qwen_API_Key
JINA_API_KEY=你的_Jina_API_Key
```

配置前端 `.env`：

```bash
cd frontend/docforge-web
cp .env.example .env
```

内容：

```dotenv
VITE_DOCFORGE_API_BASE_URL=http://127.0.0.1:8000/api
VITE_DOCFORGE_USE_MOCK=false
```

## 四、启动方式

启动 FastAPI：

```bash
.venv/bin/python -m uvicorn api.main:app --reload
```

访问健康检查：

```text
http://127.0.0.1:8000/healthz
```

启动 Vue 前端：

```bash
cd frontend/docforge-web
pnpm install
pnpm dev
```

如果 Rollup native 包报错，先确认使用的是系统 Node.js + Corepack + pnpm，再清理并重装前端依赖：

```bash
rm -rf node_modules dist .vite .cache tsconfig.tsbuildinfo
pnpm install
pnpm dev
```

访问：

```text
http://127.0.0.1:5173/
```

使用真实任务时：

```text
http://127.0.0.1:5173/?run_id=20260609_143000_ab12
```

如果没有 `run_id`，页面会显示空状态，提示如何进入任务。

## 五、如何创建或获得 run_id？

当前 v0.1 的 Vue 工作台依赖已有 `run_id`。

可以通过以下方式获得 `run_id`：

1. 使用 Streamlit 调试入口创建任务；
2. 使用已有 `data/runs/{run_id}`；
3. 等后续版本补充 Vue 创建任务入口。

启动 Streamlit 调试入口：

```bash
streamlit run app/main.py
```

然后创建任务，获得 `run_id`。

注意：Streamlit 是调试入口，不是正式产品入口。

## 六、使用内置样例体验完整流程

如果项目仍保留样例导入功能，可以通过 Streamlit 调试入口加载样例，生成 `run_id`，再用 Vue 工作台打开。

步骤：

1. 启动 FastAPI；
2. 启动 Vue 前端；
3. 可选：启动 Streamlit 调试入口创建样例任务；
4. 获得 `run_id`；
5. 打开 Vue URL：`/?run_id=xxx`；
6. 查看左侧资料；
7. 查看中间 Agent 消息；
8. 查看右侧 `prior_hint` 和诊断；
9. 触发当前主操作；
10. 下载 DOCX 或风险版 DOCX。

## 七、使用自己的资料

DocForge v0.1 支持三类资料。

外部参考软著：

- 支持 `docx` / `pdf` / `md` / `txt`。
- 只用于目录、章法、语言风格、配图方式和软著常见表达。
- 不作为产品事实来源。

自有产品资料：

- 支持 `docx` / `pdf` / `md` / `txt` / `html`。
- 可作为产品事实来源。
- 建议上传产品介绍、PRD、HLD、详细设计、功能说明、流程说明。

产品截图：

- 支持 `png` / `jpg` / `jpeg` / `webp`。
- 当前只作为展示材料登记和配图候选。
- 不做 OCR。
- 不做视觉模型解析。
- 不作为强产品事实证据。
- 不用于推断当前版本已实现功能。

## 八、Vue 工作台页面说明

左侧：资料与项目上下文。

包括：

- 当前任务；
- 参考资料；
- 自有资料；
- 截图；
- 导出历史。

中间：Agent 对话区。

包括：

- Agent 消息；
- 结构化卡片；
- 当前主操作；
- 错误提示；
- 导出结果。

右侧：运行设置与诊断区。

包括：

- 产品类型 `prior_hint`；
- 输出文档类型；
- 参考风格强度；
- 截图策略；
- 风险策略；
- 当前诊断。

特别说明：

- 右侧设置只是 `prior_hint`，不会锁死 Agent 判断。
- 只有结构化 action 才会推进 workflow。
- 前端不会直接修改 workflow 状态。

## 九、mock 模式和真实 API 模式

前端 mock 模式：

```dotenv
VITE_DOCFORGE_USE_MOCK=true
```

用途：

1. 检查界面；
2. 不需要后端；
3. 不代表真实生成结果。

真实 API 模式：

```dotenv
VITE_DOCFORGE_USE_MOCK=false
```

用途：

1. 调用 FastAPI；
2. 上传真实资料；
3. 执行真实 action；
4. 下载真实 DOCX。

## 十、输出结果在哪里？

最终用户通过 Vue 工作台下载 DOCX。

本地文件位于：

```text
data/runs/{run_id}/exports/
```

正常版：

```text
软件著作权文档.docx
```

风险版：

```text
软件著作权文档_风险版.docx
```

系统内部文件：

- `state.json`
- `evidence_map`
- `audit_report`
- `draft_v1` / `draft_v2` / `draft_v3`

这些不是正式下载产物。

## 十一、常见问题

1. 页面提示没有 `run_id` 怎么办？

   当前 v0.1 的 Vue 工作台依赖已有 `run_id`。可以先用 Streamlit 调试入口创建任务，或使用已有 `data/runs/{run_id}`，再访问 `http://127.0.0.1:5173/?run_id=<run_id>`。

2. FastAPI 启动不了怎么办？

   先确认 Python 版本为 3.11 或 3.12，并确认依赖已安装、根目录 `.env` 已存在。启动命令应在项目根目录执行：`.venv/bin/python -m uvicorn api.main:app --reload`。也可以直接使用项目标准启动脚本 `scripts/dev.sh`。

3. 前端页面打不开怎么办？

   先确认 Node.js 和 pnpm 可用，并在 `frontend/docforge-web` 下执行过 `pnpm install`。再执行 `pnpm dev`，打开终端输出的本地地址，默认是 `http://127.0.0.1:5173/`。

4. 上传失败怎么办？

   确认 FastAPI 正在运行，前端 `.env` 中 `VITE_DOCFORGE_API_BASE_URL=http://127.0.0.1:8000/api`，并确认上传文件类型属于支持范围。

5. action 提示当前状态不能执行怎么办？

   这通常说明 workflow 当前阶段还没准备好执行该操作。按页面提示先完成上一阶段，或查看右侧诊断和下一步建议。

6. 为什么截图不能作为事实证据？

   v0.1 不做 OCR 和视觉模型解析。截图只登记为展示材料和配图候选，不能用来推断按钮、菜单、页面功能或当前版本能力，避免把不可验证信息写进软著正文。

7. 能否导出 PDF？

   当前 v0.1 不导出 PDF，只导出 DOCX。

8. 能否不启动 Streamlit？

   可以。Vue 工作台不依赖 Streamlit 运行。但如果你还没有 `run_id`，当前版本通常需要通过 Streamlit 调试入口创建任务，后续版本会补充 Vue 创建任务入口。

9. 能否只用 mock 模式？

   可以用于检查界面和布局。mock 模式不调用后端，也不代表真实生成结果，不能用于验证真实资料处理或 DOCX 生成。

10. API Key 缺失怎么办？

    本地界面或流程联调可以先使用 mock provider：`DEFAULT_LLM_PROVIDER=mock`、`DEFAULT_EMBEDDING_PROVIDER=mock`。真实生成需要配置对应服务的 API Key。

## 十二、当前 v0.1 未完成能力

当前 v0.1 未完成：

1. 不做登录权限；
2. 不做多项目管理；
3. 不做 SSE；
4. `confirm-product-type` / `confirm-doc-plan` 已接入真实确认；默认无冲突时条件自动确认，有冲突或风险时使用人工确认卡片；
5. 不导出 PDF；
6. 不导出独立审计报告；
7. 不做截图 OCR；
8. 不做网页 URL 采集；
9. 不接代码仓库；
10. Streamlit 仍保留为调试入口。
