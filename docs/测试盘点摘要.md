## 当前命令定位

- `scripts/test_quick.sh`：普通开发快速基线；覆盖 API 契约、schema/state、用户可读错误、diagnostics。
- `scripts/test_frontend.sh`：前端统一验证入口；进入 `frontend/docforge-web` 并运行 `pnpm run build`。
- `scripts/test_dev.sh`：中等范围后端回归；仅在后端核心共享逻辑、多个 unit 模块、API 契约或状态模型受影响时运行。
- `scripts/test_smoke.sh`：上传、解析、人工确认、状态推进、导出等主链路变更后运行。
- `scripts/test_integration.sh`：workflow、evidence、DOCX、diagnostics、artifact 或跨模块协作变更后运行。
- `scripts/test_full.sh`：Sprint 完成、Release Candidate、测试治理变更、核心架构变更、全局状态机变更、公共 fixture 或全局 pytest 配置变更时运行。
- `scripts/test_durations.sh`：仅用于测试治理和慢测试诊断，不进入日常开发默认流程。

## 日常策略

普通开发默认不是 `scripts/test_dev.sh`，更不是 `scripts/test_full.sh`。

普通开发应先查看：

```bash
git diff --name-only
```

再根据 [docs/test_impact_matrix.md](test_impact_matrix.md) 选择受影响测试。普通开发默认先跑受影响测试文件；需要快速后端基线时才运行：

```bash
scripts/test_quick.sh
```

前端改动运行：

```bash
scripts/test_frontend.sh
```

## 分层目录

- `tests/unit`：单函数、单服务、单 agent、单状态转换。
- `tests/contract`：API 响应结构、状态枚举、`next_action`、错误码、前后端契约字段。
- `tests/smoke`：最小主链路冒烟。
- `tests/integration`：多模块协作、完整 workflow、DOCX、diagnostics、artifact。
- `tests/fixtures`：测试资料。
- `tests/helpers`：跨测试层共享 helper；smoke 不应从 integration 测试文件导入 helper。
